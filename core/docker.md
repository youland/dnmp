# 🐳 容器引擎与安全锁

> **🎯 极客箴言**：Docker 是现代运维的核武器。但如果你不懂得如何给核武器加上安全锁，它就会成为黑客轰开你服务器大门的捷径。

DNMP 的 Docker 模块不仅会为你部署最新版的容器引擎，更重要的是，它在底层为你划定了两道极其森严的“极客防线”：**本地端口锁**与**物理数据隔离区**。

---

### <span style="color: #1e90ff;">✒️ 极速拉起容器引擎</span>

在 SSH 终端中执行以下指令，系统将自动配置 Docker 官方源，并拉起整套容器底座：

```bash
bash dnmp docker
```

☕ 极客提示：此命令将全自动安装 Docker CE、Docker Compose 插件，并为你部署好 Portainer 极客可视化面板。  

**🛡️ 核心防线一：127.0.0.1 绝对端口锁（防裸奔机制）**  
> **🚨 新手踩坑预警：Docker 会无视你的防火墙！**  
很多新手在运行容器时，习惯直接映射端口，比如 -p 8080:80。你以为你没在 UFW/iptables 里放行 8080 端口就很安全？错！Docker 在底层会直接修改 iptables 规则，把你的 8080 端口强行暴露在公网上，任人爆破！

> **👑 我们的极客破局法：反向代理锁**  
在 DNMP 的架构理念中，所有的容器应用 **绝对不允许** 直接对外暴露端口。  
我们在部署任何容器时，强制要求加上 127.0.0.1: 的前缀。例如：
```
# 规范的极客 docker-compose.yml 写法
ports:
  - "127.0.0.1:8080:80"
  ```
在有公网的情况下，强烈建议使用nginx反代使用ssl证书访问。如果是内网、或者必须使用IP访问，可以去`127.0.0.1`去掉，写成`- "8080:80"`。

**🗂️ 核心防线二：/var/www/docker 数据集中隔离区**  
> **🚨 运维灾难预警：幽灵般的数据卷**  
如果你直接使用 Docker 默认的 Volume 挂载机制，你的核心业务数据会被打散并深埋在 /var/lib/docker/volumes/ 这种反人类的底层目录中。一旦服务器宕机需要重装，备份这些数据简直就是一场噩梦。

> **👑 我们的极客破局法：资产绝对掌控**  
在 DNMP 规范中，我们为你开辟了绝对隔离的物理存储区：/var/www/docker/。  
我们强烈建议你：所有的容器配置文件、挂载数据、docker-compose.yml，都必须以此目录为基准进行按目录分类。例如你的某个网盘应用：
``` /var/www/docker/
└── nextcloud/
    ├── docker-compose.yml
    ├── data/          # 挂载的物理数据目录
    └── config/        # 挂载的配置文件目录 
```

**容灾备份的极致体验：**  
当你需要整机迁移或者日常备份时，你再也不需要去寻找那些幽灵数据卷。你只需要极其潇洒地执行一行命令：
```
tar -czvf docker_backup.tar.gz /var/www/docker/
```
你的所有容器配置和核心数据，一字节不少地全量打包完成！哪怕服务器烧毁，换台新机器解压后 docker compose up -d，服务瞬间满血复活。
